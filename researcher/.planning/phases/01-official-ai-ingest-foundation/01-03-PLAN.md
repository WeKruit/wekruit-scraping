# 01-03 Plan: Crossref Metadata Backfill And Replay / Incremental Controls

## Objective
Extend the OpenAlex ingest backbone with DOI-based Crossref metadata backfill and make runs replayable and incrementally rerunnable without redefining the raw staging contract or expanding into merge/enrichment behavior.

## Files To Modify
- Create: `researcher/researcher_crossref.py`
- Modify: `researcher/researcher_pipeline.py`
- Modify: `researcher/researcher_run_state.py`
- Modify: `researcher/researcher_sources.py`
- Create: `researcher/tests/test_researcher_crossref.py`
- Create: `researcher/tests/test_researcher_pipeline_replay.py`
- Modify: `researcher/README.md`

## Tasks
1. Implement DOI-driven Crossref backfill in `researcher/researcher_crossref.py`.
   - Read DOI-bearing paper records from the staged OpenAlex raw paper file for the current run.
   - Query Crossref only as metadata backfill for those DOIs; do not use Crossref as a discovery crawler.
   - Write Crossref raw responses to `researcher/data/runs/<run_id>/crossref/papers_backfill_raw.jsonl`.
   - Record which DOI values were requested, returned, missing, or failed.

2. Add replay mode to the pipeline.
   - Allow the user to rerun a prior `run_id` in replay mode using staged raw files instead of network calls.
   - Rebuild any derived phase-1 summaries and manifests from staged raw artifacts without modifying the raw JSONL payloads.
   - Fail fast if required staged files are missing or if the run manifest is incomplete.

3. Add incremental controls for same-preset reruns.
   - Define one incremental rule set in `researcher/researcher_run_state.py` keyed by source, preset type, preset value, and since year.
   - Store checkpoint state such as OpenAlex cursor / last successful page and Crossref processed DOI offsets.
   - On incremental rerun, create a new run directory and manifest while referencing the previous successful run as the baseline.
   - Preserve semantics: new run artifacts may add records, but they must not rewrite prior raw files in place.

4. Extend `researcher/researcher_pipeline.py` for multi-source phase-1 execution.
   - Support `openalex` only, `crossref` only for an existing staged run, and `openalex + crossref` together in sequence.
   - Enforce execution order: Crossref can run only after OpenAlex raw paper staging exists for the target run.
   - Update final run status based on per-source outcomes and checkpoint persistence.

5. Add tests for Crossref, replay, and incremental behavior.
   - `test_researcher_crossref.py`: mocked DOI backfill writes raw Crossref records and preserves DOI-to-response accounting.
   - `test_researcher_pipeline_replay.py`: replay mode rebuilds summaries from staged files without issuing source requests.
   - Add coverage for incremental reruns creating a new run manifest linked to a baseline run instead of mutating old artifacts.

6. Update `researcher/README.md` with replay and incremental operator guidance.
   - Document how to run OpenAlex + Crossref in one phase-1 flow.
   - Document replay mode and when to use it.
   - Document incremental rerun semantics and the no-overwrite rule for raw staging.

## Done Criteria
- `researcher/researcher_crossref.py` backfills DOI metadata from staged OpenAlex papers and writes raw Crossref artifacts under the same run
- Pipeline replay mode can rebuild phase-1 summaries from staged raw files without network access
- Incremental reruns create a fresh run with checkpoint lineage instead of mutating prior run artifacts
- Crossref execution is gated on staged OpenAlex paper data and never becomes a discovery path
- Tests verify DOI backfill behavior, replay behavior, and incremental no-overwrite semantics
- README explains multi-source execution, replay, and incremental controls

## Risks / Dependencies
- Crossref response quality depends on DOI coverage from OpenAlex; missing DOIs are normal and must be recorded, not hidden
- Replay mode is only trustworthy if `01-01` kept raw staging append-only and source-native
- Incremental logic can silently drift into merge behavior if it deduplicates across runs; this plan must stop at source-run lineage only
