# 01-02 Plan: OpenAlex-Led Ingest And AI/ML Query Presets

## Objective
Implement the first real official-source ingest path by making OpenAlex the discovery backbone for AI/ML papers and authors, driven by deterministic concept, venue, and keyword presets that write raw source-native outputs into the phase-1 staging contract.

## Files To Modify
- Create: `researcher/researcher_openalex.py`
- Modify: `researcher/researcher_pipeline.py`
- Modify: `researcher/researcher_sources.py`
- Modify: `researcher/researcher_run_state.py`
- Create: `researcher/tests/test_researcher_openalex.py`
- Create: `researcher/tests/test_researcher_pipeline_openalex.py`
- Modify: `researcher/README.md`

## Tasks
1. Implement AI/ML preset definitions in `researcher/researcher_openalex.py`.
   - Support exactly three preset families for phase 1: `concept`, `venue`, and `keyword`.
   - Encode the initial AI/ML preset vocabulary directly and explicitly, for example:
     - concepts: artificial intelligence, machine learning, deep learning, natural language processing, computer vision
     - venues: NeurIPS, ICML, ICLR, CVPR, ACL, EMNLP
     - keywords: transformer, diffusion model, reinforcement learning, large language model
   - Keep preset expansion deterministic; do not add fuzzy matching or generic search crawling.

2. Implement OpenAlex request building and pagination.
   - Translate the validated preset input into one OpenAlex query builder with source-aware filters for preset type, publication date lower bound, and page size.
   - Preserve raw source response semantics for papers and authors; do not normalize into canonical researcher records.
   - Capture request cursor, page counts, and retry attempts into the run-state contract from `01-01`.

3. Implement paper and author extraction for raw staging.
   - Write paper records to `researcher/data/runs/<run_id>/openalex/papers_raw.jsonl`.
   - Write deduplicated author records to `researcher/data/runs/<run_id>/openalex/authors_raw.jsonl`.
   - Keep OpenAlex-native IDs, DOI values, authorship payloads, institutions, topics, and cited-by counts intact enough for later phases.
   - Add a small run summary JSON with counts for fetched papers, unique authors, with-DOI papers, and papers skipped for malformed payloads.

4. Wire OpenAlex into `researcher/researcher_pipeline.py`.
   - Add CLI execution for `--source openalex` and default source selection when no source is supplied.
   - Enforce that one run chooses exactly one preset type and one preset value.
   - Persist source completion state back into the run manifest on success or failure.

5. Add verification tests around OpenAlex ingest.
   - `test_researcher_openalex.py`: request builder returns the expected filter structure for concept, venue, and keyword presets.
   - `test_researcher_pipeline_openalex.py`: a mocked OpenAlex run writes paper/author raw JSONL files, updates manifest stats, and records cursor/checkpoint metadata.
   - Cover deduplication of repeated authors within one batch without inventing cross-source merge behavior.

6. Update `researcher/README.md` with phase-1 usage examples.
   - Add example commands for concept, venue, and keyword presets.
   - Document expected raw outputs and the fact that OpenAlex is the only discovery backbone in phase 1.

## Done Criteria
- `researcher/researcher_openalex.py` can execute official OpenAlex ingest for concept, venue, and keyword presets
- One run writes raw OpenAlex papers and authors under the run-scoped staging layout from `01-01`
- OpenAlex request / retry / cursor metadata is captured in run manifests and checkpoints
- Pipeline CLI can launch an OpenAlex ingest run with deterministic preset validation
- Tests verify query building, raw artifact emission, and manifest/checkpoint updates
- README documents concrete OpenAlex phase-1 commands and output paths

## Risks / Dependencies
- OpenAlex filter semantics for concept and venue names may drift; the executor must validate live behavior during implementation
- Keyword presets can become too broad if they are treated as free-form search; keep them to explicit AI/ML terms only
- Author extraction must remain raw-source staging, not the start of identity resolution
